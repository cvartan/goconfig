# go-config

`go-config` — лёгкий Go-пакет для доступа и управления конфигурацией приложения.

Значения конфигурации загружаются из следующих источников (в порядке приоритета):

1. Файлы конфигурации (JSON, YAML, TOML)
2. Переменные окружения ОС (имеют приоритет)
3. Файл `.env` (автоматически загружается из рабочего каталога приложения)

Поддерживается автоматическое связывание значений конфигурации с экспортируемыми полями структур.

---

## Возможности

- Конфигурационные файлы JSON, YAML, TOML
- Поддержка переменных окружения (со значениями по умолчанию)
- Автоматическая загрузка файла `.env` из рабочего каталога
- Связывание значений конфигурации с полями структур через struct tags
- Автоматическое определение типов (`string`, `bool`, `int64`, `float64`)
- Минимальный и явный API в духе Go
- Поддержка пользовательских читателей и парсеров

## Требования к конфигурационным файлам

### Общие

- Поддерживаемые типы значений: `string`, `bool`, числовой тип (при сохранении приводится к int64 или float64)
- Значения неподдерживаемых типов игнорируются
- В качестве значения можно указать ссылку на переменную окружения: `${ENV_VARIABLE:DEFAULT_VALUE}`

Тип значения определяется либо по переменной окружения, либо по значению по умолчанию.
При этом значение по умолчанию может отсутствовать (в этом случае формат `${ENV_VARIABLE}`).

Если значение параметра - строка, то можно использовать подстановку значения внутри строки:
```json
{
    "url": "${SRV_PROTOCOL:https}://${SRV_NAME}:${SRV_PORT:443}/api"
}
```

### Поддержка файла `.env`

Библиотека автоматически загружает переменные окружения из файла `.env` в рабочем каталоге приложения (если он существует).

Формат:
```
# Строки комментариев начинаются с #
DB_HOST=localhost
DB_PORT=5432
DEBUG=true
```

- Переменные окружения ОС имеют приоритет над значениями из файла `.env`
- Файл `.env` является опциональным и игнорируется без ошибок, если не найден
- Поддерживается формат `KEY=VALUE` с опциональными кавычками

### JSON и YAML

- Корневым элементом должен быть объект
- Использование массива в качестве корневого элемента не поддерживается

## Использование

### Опции конфигурации

Структура `Options` используется для настройки менеджера конфигурации:

```go
type Options struct {
    Path          string          // Путь к каталогу с файлом конфигурации
    Source        string          // Путь к конфигурации (при добавлении собственных ридеров может указываться путь не только к файлу)
    Filename      string          // Имя файла конфигурации
    Format        string          // Формат представления данных в конфигурации (по умолчанию доступны: json, yaml или toml)
    Reader        Reader          // Пользовательский читатель конфигурации (см. интерфейс Reader)
    Parser        Parser          // Пользовательский парсер конфигурации (см. интерфейс Parser)
    Context       context.Context // Контекст приложения
}
```

При передаче `nil` в `NewConfiguration` используются следующие значения по умолчанию:
- `Format`: `json`
- `Path`: текущий рабочий каталог
- `Filename`: `config.json`
- `Source`: комбинация `Path` и `Filename`

---

### Расположение конфигурационного файла

Файл конфигурации может быть задан одним из способов:

1. **По умолчанию** — `config.json` в рабочем каталоге приложения
2. **Статически** — через опции, при создании конфигурации
3. **Динамически** — через флаги приложения:

- `-config.path=/opt/app/config`
- `-config.filename=config.yml`
- `-config.format=yaml`
- `-config.source=/opt/app/config/config.yml` (при указании этого параметра, параметры config.path и config.filename не используются)

---

### Парсеры конфигурации

Три парсера доступны (зарегистрированы по умолчанию):

- JSON парсер (для `Format = "json"`)
- YAML парсер (для `Format = "yaml"`)
- TOML парсер (для `Format = "toml"`)

Вы можете создать собственный парсер (реализовать интерфейс `Parser`) и зарегистрировать его через метод `RegisterParser(format string, parser Parser)`.

## Экспортируемые типы и интерфейсы

### Configuration

Основной тип для работы с конфигурацией приложения.

```go
type Configuration struct {
    // Содержит методы для доступа к параметрам конфигурации
}
```

#### Создание конфигурации

```go
func NewConfiguration(options *Options) *Configuration
```

Создаёт новый менеджер конфигурации с указанными опциями. Передайте `nil` для использования опций по умолчанию.

#### Методы

| Метод | Описание |
|------|----------|
| `Apply()` | Загрузка и применение конфигурации |
| `Add(key string, valueType ValueDataType) *Parameter` | Добавление параметра без установки значения |
| `Set(key string, value any)` | Установка значения параметра (или добавление при его отсутствии) |
| `Get(key string) *Parameter` | Получение параметра конфигурации |
| `GetAll() []*Parameter` | Получение всех параметров |
| `Lookup(filter string) []*Parameter` | Получение массива параметров по префиксу |
| `Bind(object any)` | Привязка структуры к конфигурации |
| `GetArrayValues(key string) []any` | Получение значений массива параметров |
| `GetIntArray(key string) []int64` | Получение значений для массива параметров с целочисленными значениями |
| `GetFloatArray(key string) []float64` | Получение значений для массива параметров со значениями с плавающей точкой |
| `GetStringArray(key string) []string` | Получение массива строковых значений для массива параметров |

---

### Parameter

Тип для параметра конфигурации.

```go
type Parameter struct {
    // Содержит данные параметра с потокобезопасным доступом
}
```

#### Методы

| Метод | Описание |
|------|----------|
| `Name() string` | Получение имени параметра |
| `Value() any` | Получение raw-значения параметра |
| `Type() reflect.Kind` | Получение обобщённого типа значения параметра |
| `Int() int64` | Получение значения параметра с типом int64 |
| `Float() float64` | Получение значения параметра с типом float64 |
| `Bool() bool` | Получение значения параметра с типом bool |
| `String() string` | Получение значения параметра с типом string |
| `SetValue(value any)` | Установка значения параметра |

---

### StructuredConfiguration

Дженерик-тип для работы с типизированными структурами конфигурации.

```go
type StructuredConfiguration struct {
    C *configuration // Configuration
}
```

#### Функции

| Функция | Описание |
|--------|----------|
| `NewStructuredConfiguration[T any](options *Options) *T` | Создание типизированной структурированной конфигурации |

#### Методы

| Метод | Описание |
|------|----------|
| `Apply()` | Загрузка и применение конфигурации |

---

### Reader

Интерфейс для чтения данных конфигурации из источника.

```go
type Reader interface {
    Read(source string) ([]byte, error)
}
```

Реализуйте этот интерфейс для создания пользовательских читателей, которые могут читать конфигурацию из любого источника (файлы, HTTP, базы данных и т.д.).

---

### Parser

Интерфейс для разбора данных конфигурации.

```go
type Parser interface {
    Parse(data []byte) (map[string]any, error)
}
```

Реализуйте этот интерфейс для создания пользовательских парсеров для различных форматов данных. Зарегистрируйте пользовательские парсеры через `RegisterParser(format string, parser Parser)`.

## Привязка структур к конфигурации

Значения параметров конфигурации могут быть связаны с экспортируемыми полями структур с помощью struct tag `config`.

Правила:

- Поля должны быть экспортируемыми
- Тип поля должен соответствовать типу значения конфигурации
- Если параметр отсутствует в файле, используется переменная окружения (если она задана)

Существует два способа связывания конфигурации со структурами:

### Способ 1: Использование метода Bind

Создайте структуру с тегами `config`, затем используйте методы `Bind` и `Apply`:

```go
package main

import (
    "github.com/cvartan/goconfig"
)

type AppConfig struct {
    Name  string `config:"app.name"`
    Debug bool   `config:"app.debug"`
}

func main() {
    var cfg AppConfig

    config := goconfig.NewConfiguration(nil) // Используем опции по умолчанию

    config.Bind(&cfg) // Заполняем поля структуры
    config.Apply() // Читаем конфигурацию из файла и применяем значения параметров на атрибуты связанных структур
}
```

### Способ 2: Использование StructuredConfiguration

Создайте структуру, которая включает `goconfig.StructuredConfiguration`, затем инициализируйте её с помощью `InitStructuredConfiguration`:

```go
package main

import (
    "github.com/cvartan/goconfig"
)

type AppConfig struct {
    goconfig.StructuredConfiguration

    Name  string `config:"app.name"`
    Debug bool   `config:"app.debug"`
}

func main() {
    // Инициализируем структурированную конфигурацию с опциями (или nil для значений по умолчанию)
    cfg := goconfig.NewStructuredConfiguration[AppConfig](nil)

    cfg.Apply() // Читаем конфигурацию из файла и применяем к полям структуры

    // Доступ к значениям конфигурации напрямую через поля структуры
    // fmt.Println(cfg.Name)
    // fmt.Println(cfg.Debug)
}
```

Подход с `StructuredConfiguration` предоставляет:

- Типобезопасный доступ к конфигурации через поля структуры
- Прямой доступ к методам конфигурации через встроенный `StructuredConfiguration` (например, `cfg.Apply()`)
- Более чистую организацию кода, объединяя структуру конфигурации и поведение

- Поля структур, являющиеся срезами или массивами, не поддерживаются
